# README 스크린샷 갤러리 보강 — 설계

**작성일:** 2026-06-08
**대상:** 루트 `README.md` + `docs/screenshots/`

## 1. 목표

완성된 프론트엔드 화면을 **실제 iOS 시뮬레이터 스크린샷**으로 캡처해 루트 README에
갤러리 + 핵심 기능 요약을 추가한다. 신규 방문자가 코드를 읽지 않고도 앱을 빠르게 파악하도록.

## 2. 범위 / 제약 (확정)

- **이미지 소스:** 실제 iPhone 17 시뮬레이터 스크린샷(`xcrun simctl io ... screenshot`). 합성 목업 아님.
- **캡처 범위:** 메인 5탭 + 핵심 상세 4컷 (~9컷).
- **README 범위:** 기존 README 유지 + "주요 기능" 요약 + "스크린샷" 갤러리 섹션 추가(가벼운 보강).
- **실행 방식:** 인라인(시뮬레이터 조작은 인터랙티브, README는 단일 문서). subagent-driven/TDD 아님.
- **코드 변경 없음** — 문서/이미지만.

## 3. 캡처 대상 (~9컷)

메인 5탭:
1. `home.png` — 홈: 총 평가 자산, 진행 중 대결(내/AI 수익률), 관심 종목(API 시세), 바로가기.
2. `market.png` — 거래(마켓 대시보드): 상승/하락 카운트, 검색, 종목 리스트.
3. `duel.png` — 대결 탭(빈 상태 + 새 대결 시작) 또는 대결 설정 화면.
4. `history.png` — 내역(거래 내역): 총매수/매도 요약 + 거래 목록.
5. `account.png` — 마이(계정): 프로필, 자산 요약, 보유 종목, 메뉴.

핵심 상세:
6. `stock_detail.png` — 종목 상세: 가격, 캔들 차트 + 토글, 호가창, 매수/매도.
7. `portfolio_detail.png` — 포트폴리오 상세(AI): 보유 종목 + 따라사기 버튼.
8. `ai_feedback.png` — AI 피드백(목록 또는 상세).
9. `stock_info.png` — 종목 상세정보(52주·거래량·시총·PER/PBR).

(실제 캡처 가능 여부에 따라 8~9컷으로 가감; 최종 목록은 README 갤러리와 일치시킨다.)

## 4. 캡처 방법

1. iPhone 17 시뮬레이터 부팅 + 백엔드(`:8080`, local 프로파일) 기동 + `flutter run`
   (홈 관심종목·마켓·종목상세가 API 시세로 표시되도록).
2. 시뮬레이터를 조작(하단 5탭 이동, 리스트 항목/액션 탭)하여 각 화면 진입.
3. 각 화면을 `xcrun simctl io <udid> screenshot docs/screenshots/<name>.png`로 캡처.
4. 캡처마다 의도한 상태(데이터 로드 완료 등)인지 확인 후 다음 화면.

## 5. 저장 / 표시

- 저장: `docs/screenshots/*.png` (repo 커밋).
- README 표시: 3열 마크다운 표 그리드, 폭 제어 위해 HTML img —
  `<img src="docs/screenshots/home.png" width="240">` + 한글 캡션.

## 6. README 변경 (루트 `README.md`)

- 상단(제목·태그라인·`## 구조`·`## 실행`·`## 문서`) 유지.
- 신규 `## 주요 기능`: AI 투자 대결 · 모의 시세 마켓 · 종목 상세(캔들/호가/상세정보) ·
  포트폴리오·따라사기 · AI 피드백/심리 · 거래내역/배당/환전/요금제.
- 신규 `## 스크린샷`: 3열 표 그리드(위 이미지 + 캡션).
- 그 외 텍스트는 surgical 유지.

## 7. 검증

- `docs/screenshots/`에 캡처 파일 존재, README 상대경로가 실제 파일명과 일치.
- 마크다운 렌더 시 깨진 이미지 링크 없음.
- 사용자에게 완성본을 보여주고 피드백 수집 → 반영.
