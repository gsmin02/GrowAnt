# AI 3종 정의 (명칭·역할 분리)

용어 혼동을 막기 위해 'AI'를 세 시스템으로 분리한다.

## 1) 대결 AI (전략 엔진)
- 역할: 사용자와 수익률을 겨루는 상대.
- 동작: 성향 프롬프트(yml 관리) → 전략 생성 → 백테스트 → 사용자가 난이도/전략 선택 → 기간 동안 자동 매매.
- 매매 자체엔 LLM 미개입(전략 조건 충족 시 함수 호출). 9주차 흐름.
- 성향은 **연구 대상**: yml로 프롬프트를 교체·실험하고, 의사결정으로 선택된 성향을 적용.

## 2) AI 피드백
- 역할: 거래 후 LLM 피드백(잘한점/못한점/제안).
- 모델: Gemini **Flash**(Free·Standard) / **Flash·Pro**(Premium). 추후 멀티벤더.
- 포트: `AiClient`.

## 3) 현재 심리 예측  (구 'AI 심리 리포트')
- 역할: 최근 매매 기반 투자 심리·성향 분석/예측.
- **연구 필요**: Cowork 기반 연구 프로세스로 베이스라인 → 개선.

## 시세 데이터
- 현재: 시뮬레이션(`MarketDataProvider = sim`). KIS는 동일 포트로 후속 교체.
